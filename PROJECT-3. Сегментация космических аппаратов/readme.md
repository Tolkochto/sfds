# Сегментация космических аппаратов

## О проекте
Данный проект разработан в рамках задачи по компьютерному зрению (CV) для стартапа в сфере космических технологий. Главная цель — построение модели семантической сегментации действующих космических объектов (искусственных спутников и станций) для предсказания их траектории и избежания столкновений с космическим мусором на орбите.

Целевая метрика качества модели: **mIoU > 70%** на валидационной выборке.

## Данные
Для обучения используется набор данных **"A Spacecraft Dataset for Detection, Segmentation and Parts Recognition"** (июнь 2021 года).
*   **Объем:** 3 116 изображений (синтезированные и реальные кадры).
*   **Разрешение:** Исходное 1280x720 (в пайплайне масштабируется до 512x512 для оптимизации вычислений).
*   **Классы сегментации:**
    *   `0` — Фон (Background)
    *   `1` — Корпус (Body) 
    *   `2` — Солнечные панели (Solar Panels) 
    *   `3` — Антенны (Antennas)

*Выборки строго разделены согласно индексации файлов:*
*   **Train:** Индексы 0-402 (точные маски) + 1003-3116 (грубые маски).
*   **Val:** Индексы 403-1002 (точные маски).

## Технологический стек
*   **PyTorch** & **PyTorch Lightning** — фреймворк и высокоуровневая обертка для создания чистого и структурированного цикла обучения.
*   **DeepLabV3 (ResNet50)** — мощная архитектура сегментации, инициализированная предобученными весами ImageNet (Transfer Learning).
*   **Albumentations** — библиотека для пространственных и цветовых аугментаций на лету (Affine, RGBShift, RandomBrightnessContrast).
*   **Torchmetrics** — надежный подсчет метрики Jaccard Index (mIoU).
*   **OpenCV** & **Matplotlib** — обработка, цветовая фильтрация и визуализация данных.

## Особенности реализации
1.  **Оптимизированный DataLoader:** Изменение порядка операций (сначала `cv2.resize`, затем цветовая фильтрация матриц) позволило многократно ускорить подготовку батчей, устранив серьезный CPU-bottleneck.
2.  **Умный парсинг датасета:** Класс `Dataset` автоматически сканирует директории и сопоставляет пары «изображение-маска» по базовому имени файла, динамически игнорируя системные артефакты и поврежденные файлы без пар.
3.  **Защита от переобучения:** Использование коллбэков `EarlyStopping` (остановка при стагнации `val_loss`) и `ModelCheckpoint` (сохранение лучших весов по максимальному `val_mIoU`).

## Результаты
Благодаря использованию Transfer Learning, аугментаций и корректному маппингу данных, модель уверенно преодолевает бейзлайн **mIoU > 0.70** уже на первых эпохах обучения. Архитектура DeepLabV3 с Atrous-свертками демонстрирует высокую точность в распознавании даже сложных и тонких геометрических элементов спутников (антенн).

В ноутбуке также реализована функция `visualize_predictions()` для наглядного сравнения предсказаний обученной модели с Ground Truth на валидационном батче.
